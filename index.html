Programación Orientada a Objetos (POO)

Es un paradigma de programación que organiza el código en "objetos", que son instancias de clases. Cada objeto tiene atributos (propiedades) y métodos (funciones), y se centra en representar elementos del mundo real.

Trabajando con JSON en JavaScript

Convertir un objeto en JSON:

let objeto = { nombre: "James", edad: 30 };
let jsonString = JSON.stringify(objeto); // Convierte el objeto a JSON (string)

Convertir JSON a un objeto:

let jsonString = '{"nombre": "James", "edad": 30}';
let objeto = JSON.parse(jsonString); // Convierte el JSON a un objeto

Pilares de la Programación Orientada a Objetos

  • Encapsulamiento: Agrupa los datos y las funciones que operan sobre esos datos en una clase.
  • Herencia: Permite crear nuevas clases basadas en clases existentes.
  • Polimorfismo: Habilidad de los objetos de una clase derivada de comportarse de manera diferente según su contexto.
  • Abstracción: Simplifica la complejidad ocultando detalles innecesarios.

Polimorfismo

El polimorfismo permite que un mismo método se comporte de distintas maneras dependiendo del objeto que lo invoque.

Tipos de polimorfismo:

  • Polimorfismo de Sobrecarga: Permite tener múltiples métodos con el mismo nombre pero diferentes parámetros.
  • Polimorfismo de Sobreescritura: Permite que una clase derivada modifique el comportamiento de un método de la clase base.

Relaciones entre clases en UML

  • Asociación: Es una relación simple entre dos clases.
  • Agregación: Una relación "tiene una", donde una clase contiene a otra.
  • Composición: Una forma más fuerte de agregación. Si la clase contenedora se destruye, las contenidas también.
  • Herencia: Una clase hereda de otra.
  • Relación simple: Relación básica entre dos clases.

Definiciones

  • Clase: Es el plano o molde a partir del cual se crean objetos. Contiene atributos y métodos.
  • Instancia: Es un objeto creado a partir de una clase.
  • Atributos: Son las características o propiedades de una clase u objeto.
  • Métodos: Son las funciones o comportamientos que tiene un objeto.
  • Herencia: Es la capacidad de una clase para heredar atributos y métodos de otra clase.
  • Visibilidad: Controla el acceso a los atributos y métodos de una clase (público, privado, protegido).
  • Abstracción: Simplifica la complejidad ocultando los detalles internos.
  • Objeto: Es una instancia de una clase, un ejemplar que tiene atributos y métodos propios.
  • Encapsulamiento: Consiste en ocultar el estado interno del objeto y proteger su información.
  • JSON (JavaScript Object Notation): Es un formato de intercambio de datos que utiliza pares clave-valor.
